# Bug Report Plan - Batch 2

This plan outlines 4 additional critical bugs discovered in `icode` affecting Bash Command Validation and Sandbox Workspace Directory Checks.

| ID | Issue Description | File Path | Proposed Fix |
|:---|:---|:---|:---|
| 1 | **Bash multi-command validation bypass** <ul><li>`extract_first_command` only extracts the initial alphanumeric token.</li><li>Fails to parse logical operators (`;`, `&&`, `\|\|`, `\|`, `\n`).</li><li>Payload `ls && rm -rf /` parses as `ls` and executes the destructive component.</li></ul> | `runtime/src/bash_validation.rs`, `runtime/src/permission_enforcer.rs` | Instead of analyzing just the first part of the string, properly split bash operators and validate EVERY command string component. Alternatively, block execution if suspicious shell separators are found in `ReadOnly` mode. |
| 2 | **Sudo parameter value injection** <ul><li>`extract_sudo_inner` incorrectly determines arguments to `sudo`. It attempts to skip flags (`-u`) but wrongly assesses the flag's value (e.g. `root`) as the inner command.</li><li>Payload: `sudo -u root malicious_command` validates "root" as safe and ignores the `malicious_command`.</li></ul> | `runtime/src/bash_validation.rs` | Scan the entire sequence of arguments inside the bash string after `sudo`. If ANY token is an explicitly prohibited `WRITE` or `DESTRUCTIVE` command, immediately block execution. |
| 3 | **Bash single-quote evasion flaw** <ul><li>`find_end_of_value` considers `\'` as an escaped character inside `'... '` strings, which is incorrect in Unix bash (single quotes do not support escapes).</li><li>Results in early parse-fail termination (`None`), triggering a `String::new()` empty return that bypasses further restriction validation pipelines.</li></ul> | `runtime/src/bash_validation.rs` | Correct parsing rule: disable backslash escape tracking `b'\\'` whenever the tracked quote character is a single quote (`b'\''`). |
| 4 | **Symlink traversal escape via textual fallback** <ul><li>When checking if a file bounds to `/workspace/`, if the file does not exist, canonicalization inevitably fails.</li><li>It dangerously drops back to `normalize_path` textual string comparison without checking parent directory constraints.</li><li>Writing to a symlinked `/workspace/symlink_dir/new_file` escapes bounded containment if `symlink_dir` points to `/etc/`.</li></ul> | `runtime/src/permission_enforcer.rs` | When `canonicalize()` fails, fall back to checking `parent().canonicalize()` just like `batch_edit.rs` attempts to do. String truncation rules alone without querying the `fs` for intermediary symlinks are flawed. |

## Next Steps
These 4 bugs affect the exact same pipeline components addressed in the initial evaluation. User approval is requested to start implementing the fixes according to these structural remediations.
